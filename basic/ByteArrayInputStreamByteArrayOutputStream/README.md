### 1) Тема: Работа с байтовыми потоками в памяти (ByteArrayInputStream и ByteArrayOutputStream)

2) Примеры:  
- **ImageToFrame.java:** Чтение изображения из файла, преобразование в байты и запуск отображения.  
- **ImageFrame.java:** Десериализация байтов в изображение и отрисовка в JFrame.  
- **StringCapitalizeWithByteArrayStream.java:** Преобразование строки в байты, чтение по символам и капитализация.  
- **ImageToByteArray.java:** Загрузка изображения в байты для дальнейшего использования.

3) Объяснение темы:  
Байтовые потоки в памяти позволяют обрабатывать данные как последовательный поток байтов напрямую в оперативной памяти, без взаимодействия с файловой системой или внешними устройствами. Это обеспечивает высокую скорость для временного хранения, манипуляции или передачи бинарных и текстовых данных, таких как изображения или строки, минимизируя накладные расходы на I/O-операции. Такие потоки особенно полезны в сценариях, где данные динамически генерируются или модифицируются, например, перед отправкой по сети или отображением в GUI.

4) Основные методы и особенности:  
- **ByteArrayOutputStream.write(int b) / write(byte[] b, int off, int len):** Добавление отдельных байтов или порций массива в внутренний буфер; метод toByteArray() возвращает полный байтовый массив для извлечения данных.  
- **ByteArrayInputStream.read() / read(byte[] b, int off, int len):** Последовательное чтение байтов в int или в буфер; возвращает -1 при достижении конца потока, поддерживая циклы для полной обработки.  
- **reset() / size():** Сброс позиции чтения/записи и получение текущего размера буфера, что позволяет переиспользовать поток без создания нового.  
- Особенности: Автоматическое расширение буфера при переполнении; совместимость с ImageIO для форматов вроде JPG; обработка кодировок (например, UTF-8 для текста); не требует закрытия, так как работает в памяти, но рекомендуется в try-with-resources для consistency.
